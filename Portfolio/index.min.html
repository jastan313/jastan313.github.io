|DOCIT|

|DOCIT| is a clean, single-page web application for anonymous-based creative writing and reading. Writers (under an Alias handle) can brainstorm and draft any type of text-based writing (e.g. poetry, lyrics, short stories, prose, narratives, etc); finalized drafts can be published to the public feed where other writers may read, rate, and write notes, expressing their thoughts (e.g. criticism, analysis, and/or any constructive feedback of the work). As a solo project, I developed |DOCIT| utilizing the MEAN stack with a keen focus on a simple, responsive design to emphasize the value from writers' works while providing a pleasing, paper-esque aesthetic.

MongoDB, AngularJS, ExpressJS, NodeJS, HTML/CSS

Fractalize

Fractalize is an escape-time fractal generator website. In real-time, Fractalize uses an iterative coloring process which determines how divergent a pixel point is depending on the recurrence relation for the specific fractal type based on user inputed values. Users may play around with sensitive variables to modify the generation, explore fractals in depth by moving and zooming the canvas, and share their discoveries by sharing the corresponding link or image file. Independently, I researched and incorporated the mathematical formulas that create these intricate, colorful designs in JavaScript.

JS, HTML/CSS

TScorify

TScorify, a Spark application written in Scala, takes a text file dataset and associates each individual file with its keywords. Keywords are selected based on their TF-IDF score. It is accompanied by the TScorify website to display its resulting data of keyword scores in a graphical context. TScorify was developed by myself as a introductory gateway into Spark technologies. I implemented the algorithms necessary to manipulate Spark RDDs and researched how to design a TD-IDF scorer in order to produce the results desired. The website portion is a minimalistic site built from HTML/CSS and vanilla JS for data parsing and HTML code generation.

Spark, Scala, JS, HTML/CSS, MapReduce

GlassMADA

GlassMADA (Memory Assistive Display for Persons with Alzheimer's) is a Google Glass application that supports the users by retrieving lost personal information. The app functions to bring back independence to its user in a therapeutic manner. The app satisfies its objective by performing person recognition (name, relationship status, and other desired information) through passive QR scanning and by displaying past daily information derived from passive, contextually-aware photo-taking. GlassMADA is comprised of three parts: GlassMADA (Google Glass application), the Flask server, and MADA Timeline (the web application). I was responsible for designing the RESTful API for communication between the three components. In addition, I worked independently on the MADA Timeline (mostly vanilla JS and HTML/CSS), implementing a JSON parser and bit64 decoder for the contextual data requested from the server and developing the Timeline's UI to display this information.

GoogleGlass Android, JS, HTML/CSS

Battle Blocks

Battle Blocks is a physics-simulated “sandbox”, 4-player MOBA game where players build and personalize combating robots block by block. Players build their robot and fight in a literal sandbox in a King of the Hill style game mode. Players within the Money Zone (the hill) are awarded Lunch Money which can be used to purchase additional blocks to upgrade their robot. Buyable blocks include: melee weapons such as maces, ranged weapons such as cannons, new wheels for increased manuverability, and building blocks which increase robot health and grant additional effects. And of course, in combat, these appended blocks do fall off upon receiving critical damage! Built from scratch, the game consists of about 20,000 lines of C++ code, ranging from Bullet physics engine to openGL, 3D models created in Blender, and soundbytes from SFML. My partner and I worked on the entire server-side (includes server-side network, game logic, and game physics) together. Examples of a few specific features I worked on: developing custom packets to send game data to client for graphics rendering, designing and creating game event APIs to communicate with the client, implementing collision events and constraint system (orientation of blocks and breaking off blocks), and designing and implementing the damage system.

C++, Systems Design, Game Design

KJ

KJ is a 3-stage, pipelined hardware design for an 8-bit processor, optimized to run three programs with constant one clock per instruction performance. The processor design includes an unique 8-bit instruction set architecture, eight general data registers (plus an one-bit overflow register), two pipeline registers (between the three stages), asynchronous data reading, and data forwarding units (one data forwarding unit and one control unit) to avoid data/control hazards. My partner and I designed and implemented KJ in Altera Quartus II software using SystemVerilog then manually tested KJ's program execution in Altera ModelSIM by demonstrating correct operations via timing diagrams (at each clock cycle).

SystemVerilog, Hardware Architecture

Sales135

Sales135 is a retail web application where the owner posts retail products and tracks sales by running data analytics queries. A few queries include: best-selling products in a specific state, customers and their purchase totals in alphabetical order in a specific state, and total sales for a specific state. Query execution time has been minimized by two optimization techniques: column indexing and precomputed tables. Sales135 is comprised of a JSP application and a PostgreSQL database. With two other partners and a focus on the project's data analytics features, we designed the appropriate schema, implemented the database tables and corresponding queries, and introduced column indexing and precomputed tables after testing their performance improvements. .

JSP, PostgreSQL, Database Optimization
