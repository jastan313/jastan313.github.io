TScorify

by Jason Tan

TScorify, a Spark application written in Scala, takes a text file dataset and associates each individual file with its keywords. Keywords are selected based on their IF-IDF score. This website is a guide and a tool to visually display the resulting keywords.

How To Start:

(0) [Setup step; you may skip to step (1) if you already have Spark and Scala installed.] This app requires you to have Apache Spark and Scala installed on you computer. Click here, Spark and Scala, to download them respectively.

For Spark, select Spark release: "1.6.1", package type: "Pre-built for Hadoop 2.6 and later", download type: "Direct Download", and click the TGZ link to start your download.
For Scala, click the first link for Scala binaries.

(1) Click here to download the JAR containing the application.


(1.1) [Optional step; you may skip to step (2) if you do not require an example input dataset.] Click here to download a zip file containing an example directory of text files which you may use as input for testing the app.


(2) Open up command prompt and navigate to the directory containing the JAR file, tscorify.jar. [For example, if tscorify.jar is in your Desktop directory, navigate to "/Desktop/".]


(3) Type the command below:

You may choose which input and output directories to use as command line arguments. For example, if the directory containing the text files are located in a directory named "textfiles", you can use the option --input=textfiles to specify that directory as your input directory. Use the --help option for further assistance. Note: You may see an error occur with the ShutdownHookManager after the application finishes executing. This is totally fine.

YOUR_SPARK_HOME/bin/spark-submit --class "TScorify" --master local[4] tscorify.jar --verbose

(4) Navigate to the output directory and you may manually inspect the resulting data found in files named part-XXXXX. [For example, if the output directory is "/TClassify_Result/", you may find files named part-00000 and part-00001.]


(4.1) [Optional step; you may skip to step (5) if you do not require an example result dataset.] Click here to download a zip file containing an example directory of result files which you may use as input for step (5).


(5) To see the result data in a more graphical context, the result files.
